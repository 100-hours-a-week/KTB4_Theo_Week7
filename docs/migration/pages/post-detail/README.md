## 범위 판단

요청된 여섯 문서를 모두 유지한다. 상세 화면은 한 페이지 안에 다음 기능이 함께 있기 때문에 정상 시나리오, 오류 계약, 단계별 구현 경계를 분리해 기록할 필요가 있다.

- 게시글과 댓글 조회
- 이미지 갤러리
- 좋아요 토글
- 댓글 작성·수정·삭제
- 게시글 수정 이동·삭제
- 공통 Header와 로그아웃

1차 구현은 데이터 변경 위험이 없는 상세·갤러리·댓글 조회까지다. 좋아요와 삭제, 댓글 변경은 같은 문서에 계약을 기록하되 후속 단계로 구현한다. 화면 캡처는 필수가 아니며 `ui-baseline.md`를 기본 비교 자료로 사용한다.
