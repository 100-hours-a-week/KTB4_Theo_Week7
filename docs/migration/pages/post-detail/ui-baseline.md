# 게시글 상세 텍스트 UI 기준선

## 1. UI-POST-DETAIL-01 기본 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ [뒤로]                 </> 코드 한입               (프로필) │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [♥ 12 좋아요]  게시글 제목                                   │
│                (작성자) 닉네임  2026-07-19 12:34:56 (수정됨) │
│                                     조회수 100  댓글 2        │
│                                              [수정] [삭제]    │
│                ┌────── 첨부 이미지 갤러리 ──────┐           │
│                │ [이전]                  [다음] │  1 / 2     │
│                └───────────────────────────────┘             │
│                게시글 본문                                   │
│                ─────────────────────────────────             │
│                댓글                                          │
│                [댓글을 남겨주세요!]                 [등록]   │
│                (댓글 카드 목록)                              │
└──────────────────────────────────────────────────────────────┘
```

콘텐츠 최대 너비는 1,240px이며 기본 화면에서는 좌측 128px 영역에 세로 좋아요 통계를 두고 본문과 댓글을 정렬한다.

## 2. UI-POST-DETAIL-02 로딩과 오류

```text
최초 로딩: Header만 보이고 상세 article과 댓글 section은 hidden
조회 오류: alert 표시, 상세·댓글은 계속 hidden
별도 spinner·skeleton·재시도 버튼 없음
```

## 3. UI-POST-DETAIL-03 작성자 여부

```text
author=true  → [수정] [삭제]
author=false → 두 버튼 모두 미표시
edited=true  → 작성일 옆 (수정됨)
```

## 4. UI-POST-DETAIL-04 숨김 게시글

```text
제목: 숨김 처리된 게시글입니다.
본문: 숨김 처리된 게시글입니다.
좋아요: 미표시
이미지: 미표시
댓글 전체: 미표시
```

## 5. UI-POST-DETAIL-05 갤러리

```text
0개 → 갤러리 미표시
1개 → 이미지만 표시, navigation·indicator 미표시
2개 이상 → 이전·다음 버튼과 "현재 / 전체" 표시
첫 이미지 → 이전 disabled
마지막 이미지 → 다음 disabled
```

갤러리 높이는 기본 360px, 640px 이하에서는 280px이다. 이미지는 `object-fit: contain`으로 표시한다.

## 6. UI-POST-DETAIL-06 댓글

```text
(프로필) 닉네임  작성일                         [수정] [삭제]
           댓글 내용

삭제 댓글 내용 → 삭제된 댓글입니다.
댓글 없음 → 별도 문구 없음
```

작성자이면서 삭제·탈퇴 상태가 아닌 댓글에만 버튼을 표시한다.

## 7. UI-POST-DETAIL-07 댓글 폼

```text
등록 모드: textarea 빈 값, 취소 미표시, [비활성: 댓글 등록]
유효 입력: [활성: 댓글 등록]
수정 모드: 기존 내용, [취소] [활성: 댓글 수정]
오류: textarea 아래 붉은 helper
```

## 8. UI-POST-DETAIL-08 삭제 모달

```text
┌─────────────────────────────────┐
│ 게시글을 삭제하시겠습니까?      │
│ 삭제한 내용은 복구할 수 없습니다.│
│          [취소] [확인]           │
└─────────────────────────────────┘
```

댓글 대상이면 제목만 `댓글을 삭제하시겠습니까?`로 바뀐다.

## 9. 반응형 기준

| viewport | 주요 변화 |
|---:|---|
| 900px 초과 | 좌측 세로 좋아요, 본문·갤러리·댓글 128px 들여쓰기 |
| 900px 이하 | 좋아요가 상단 가로형, 들여쓰기 제거, 제목 32px |
| 640px 이하 | Header 70px, 제목 26px, meta와 댓글 header 세로 배치, 갤러리 280px |

## 10. React 비교 판정 기준

- 같은 응답에서 텍스트, 수치, 권한 버튼과 숨김 범위가 일치한다.
- 이미지 개수와 index에 따른 navigation 상태가 일치한다.
- 본문·댓글 줄바꿈과 fallback 이미지 동작이 일치한다.
- 댓글 등록·수정 모드와 helper 위치가 일치한다.
- 삭제 대상에 따라 모달 제목이 일치한다.
- breakpoint별 좋아요·meta·댓글 배치가 일치한다.
