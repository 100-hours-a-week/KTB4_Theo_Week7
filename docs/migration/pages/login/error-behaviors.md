# 로그인 오류 상태별 화면 동작

## 클라이언트 검증 오류

| 대상 | 조건 | 표시 위치 | 표시 내용 | API 요청 |
|---|---|---|---|---|
| 이메일 | 빈 값으로 blur 또는 제출 | 이메일 helper | `* 이메일을 입력해주세요.` | 보내지 않음 |
| 이메일 | 이메일 형식이 아님 | 이메일 helper | `* 올바른 이메일 주소 형식을 입력해주세요. (예: example@adapterz.kr)` | 보내지 않음 |
| 비밀번호 | 빈 값으로 blur 또는 제출 | 비밀번호 helper | `* 비밀번호를 입력해주세요.` | 보내지 않음 |
| 비밀번호 | 형식이 유효하지 않음 | 비밀번호 helper | 8~20자 및 대·소문자, 숫자, 특수문자 포함 안내 | 보내지 않음 |

폼이 유효하지 않으면 로그인 버튼이 비활성화된다.

## API 오류

API 오류를 처리하기 직전에 이메일과 비밀번호 helper를 모두 비운다.

| HTTP 상태 | `message` | UI 동작 | 페이지 이동 |
|---:|---|---|---|
| 400 | `invalid_request` | `입력값을 다시 확인해주세요.` alert | 없음 |
| 400 | `invalid_email_format` | 이메일 helper에 올바른 이메일 형식 안내 | 없음 |
| 400 | 그 외 | `입력값을 다시 확인해주세요.` alert | 없음 |
| 401 | `invalid_credentials` | 비밀번호 helper에 `* 아이디 또는 비밀번호를 확인해주세요.` 표시 | 없음 |
| 401 | 그 외 | `아이디 또는 비밀번호를 확인해주세요.` alert | 없음 |
| 429 | `too_many_login_attempts` | `로그인 시도가 너무 많습니다. 잠시 후 다시 시도해주세요.` alert | 없음 |
| 429 | 그 외 | 동일한 로그인 시도 제한 alert | 없음 |
| 500 | 구분하지 않음 | `서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.` alert | 없음 |
| 그 외 | 구분하지 않음 | `로그인 중 오류가 발생했습니다. 다시 시도해주세요.` alert | 없음 |

## 네트워크 및 응답 형식 오류

현재 구현에서 다음 오류는 HTTP 상태가 없는 일반 오류로 처리된다.

- 네트워크 연결 실패
- JSON으로 파싱할 수 없는 응답
- 성공 응답에 `data.accessToken`이 없음

이 경우 콘솔에 오류를 기록하고 다음 alert를 표시한다.

```text
로그인 중 오류가 발생했습니다. 다시 시도해주세요.
```

## 오류 후 상태 복구

성공 여부와 관계없이 요청이 끝나면 다음 동작을 수행한다.

- `isLoginSubmitting`을 `false`로 변경한다.
- 현재 입력값의 유효성에 따라 로그인 버튼을 다시 활성화하거나 비활성화한다.
- 사용자가 입력한 이메일과 비밀번호는 지우지 않는다.

## React 이전 시 주의사항

- 클라이언트 검증 오류와 서버 오류를 같은 상태 객체에 저장하더라도 기존 표시 위치를 유지한다.
- alert를 Toast 등으로 임의 변경하지 않는다. UX 변경은 마이그레이션 완료 후 별도 작업으로 다룬다.
- 401 로그인 실패를 전역 인증 만료 처리로 보내지 않는다.
- 로그인 API에는 토큰 재발급 인터셉터를 적용하지 않는다.
- 요청이 끝났을 때 입력값을 초기화하지 않는다.
