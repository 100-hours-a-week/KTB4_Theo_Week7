# 게시글 수정 정상 작동 시나리오

## 1. 기준 소스

- 화면: `pages/edit-post.html`
- 수정 동작: `js/edit-post.js`
- Header 동작: `js/profile-menu.js`
- 검증: `js/validation.js`
- 공통 요청·이미지 처리: `js/common.js`
- 스타일: `css/common.css`, `css/edit-post.css`

## 2. 페이지 정보

- 기존 URL: `pages/edit-post.html?postId={id}`
- React 예정 URL: `/posts/:postId/edit`
- 진입 대상: 로그인 사용자
- 뒤로가기: `/posts/:postId`
- 로고 이동: `/posts`
- 수정 성공: `/posts/:postId`

## 3. 화면 구성

1. 상세 뒤로가기, 서비스 로고, 프로필 메뉴가 있는 Header
2. `Edit Post`, `게시글 수정`, 설명 문구가 있는 heading
3. 기존 게시글 조회 전에는 숨겨진 수정 card
4. 기존 제목과 내용이 채워진 입력 필드
5. 선택적으로 표시되는 기존 이미지 grid
6. 새 다중 이미지 선택 input과 미리보기
7. card 하단 중앙의 `수정하기` 버튼

## 4. Route ID와 초기 조회

1. Route param `postId`를 숫자로 변환한다.
2. 숫자가 아니거나 양의 정수가 아니면 `잘못된 게시글 주소입니다.` alert 후 `/posts`로 이동한다.
3. 보호 Route가 인증을 초기화한다.
4. 같은 ID의 중복 조회를 막고 `GET /posts/{postId}`를 요청한다.
5. 조회 중에는 수정 form을 표시하지 않는다.
6. 성공하면 제목·내용과 유효한 `imageUrls`를 초기 상태로 저장한다.
7. 기존 이미지 렌더링을 마친 뒤 form을 표시한다.
8. 조회가 끝나면 폼 유효성과 pending 상태에 따라 수정 버튼을 갱신한다.

## 5. 기존 값과 검증

- 제목은 `post.title || ''`, 내용은 `post.content || ''`로 채운다.
- `imageUrls`가 배열이면 falsy 항목을 제거하고, 아니면 빈 배열로 처리한다.
- 제목은 trim 후 1~26자여야 한다.
- 내용은 trim 후 비어 있지 않아야 하며 현재 최대 길이 제한은 없다.
- 입력 중에는 버튼 유효성만 갱신한다.
- blur 시 해당 helper를 표시하거나 지운다.
- submit 시 제목과 내용을 모두 다시 검증한다.
- 조회·수정 요청 중에는 수정 버튼을 비활성화한다.

## 6. 기존 이미지 표시

- 기존 이미지가 없으면 `기존 이미지` section 전체를 숨긴다.
- 이미지가 있으면 응답 순서대로 grid에 표시한다.
- 각 경로는 공통 `resolveImageUrl()`로 브라우저 URL로 변환한다.
- alt는 `기존 게시글 이미지 {순번}`이다.
- 기존 이미지 로드가 실패하면 해당 `<img>`만 제거한다.
- 로드 실패로 모든 이미지가 제거되어도 Vanilla는 section heading을 다시 숨기지 않는다.

## 7. 새 이미지 선택

### 선택하지 않음

- `새 이미지를 선택하지 않으면 기존 이미지가 유지됩니다.`를 표시한다.
- PATCH payload에 기존 `imageUrls` 배열을 그대로 사용한다.

### 하나 이상 선택

1. 이전 새 이미지 Object URL을 모두 반환한다.
2. 새 FileList를 배열로 저장한다.
3. `{파일명들} 선택됨 - 기존 이미지를 교체합니다.`를 표시한다.
4. 선택 순서대로 새 미리보기 이미지를 표시한다.
5. PATCH payload에는 새 파일의 `name` 배열만 사용한다.
6. 기존 이미지와 새 이미지를 합치지 않고 전체 교체한다.

선택을 다시 비우면 새 미리보기를 제거하고 기존 이미지 유지 상태로 돌아간다. 페이지를 떠날 때 남은 Object URL을 모두 반환한다.

## 8. 게시글 수정 요청

1. 이미 수정 요청 중이면 추가 submit을 무시한다.
2. 제목과 내용을 다시 검증하고 실패하면 요청하지 않는다.
3. 제목과 내용의 앞뒤 공백을 제거한다.
4. 새 파일이 있으면 파일명 배열, 없으면 기존 이미지 배열을 `imageUrls`로 사용한다.
5. 버튼을 비활성화하고 `PATCH /posts/{postId}`를 한 번 요청한다.
6. 성공 응답 body는 사용하지 않고 `/posts/:postId`로 이동한다.
7. 실패하면 입력값·기존 이미지·새 파일을 유지하고 오류 UI를 표시한다.
8. 이동하지 않은 경우 pending을 해제하고 버튼을 복구한다.

## 9. 공통 Header

- 뒤로가기는 현재 게시글 상세로 이동한다.
- 로고는 게시글 목록으로 이동한다.
- 현재 사용자 프로필과 공통 회원 메뉴를 표시한다.
- 로그아웃 성공 시 `/login`으로 이동한다.

## 10. React 마이그레이션에서 보존할 동작

- Route ID 검증과 잘못된 주소 처리
- 초기 GET 완료 전 form 미표시
- 기존 제목·내용·이미지 초기화
- 기존 이미지 개별 로드 실패 제거
- 새 이미지 미선택 시 기존 배열 유지
- 새 이미지 선택 시 전체 이미지 배열 교체
- Object URL 변경·unmount cleanup
- 작성 화면과 동일한 제목·내용 검증
- GET과 PATCH 각각의 status/message 오류 처리
- 조회·수정 중 중복 요청 방지
- 수정 성공 후 현재 상세 Route 이동
