# 게시글 수정 텍스트 UI 기준선

## 1. 전체 배치

```text
┌──────────────────────────────────────────────────────────┐
│ [← 상세]              </> 코드 한입          [프로필]   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│                       Edit Post                          │
│                      게시글 수정                         │
│       작성한 내용을 다듬고 이미지 구성을 다시 정리...   │
│                                                          │
│  ┌──────────────── 수정 card ────────────────────────┐  │
│  │ 제목*                                              │  │
│  │ [기존 제목                                      ] │  │
│  │ [제목 helper]                                      │  │
│  │                                                    │  │
│  │ 내용*                                              │  │
│  │ [기존 내용 textarea                              ] │  │
│  │ [내용 helper]                                      │  │
│  │                                                    │  │
│  │ 기존 이미지                                       │  │
│  │ [기존 1] [기존 2] [기존 3]                        │  │
│  │                                                    │  │
│  │ 이미지 [파일 선택]                                │  │
│  │ 새 이미지를 선택하지 않으면 기존 이미지 유지      │  │
│  │ [새 1] [새 2] [새 3]                              │  │
│  │                                                    │  │
│  │                   [수정하기]                       │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

GET 완료 전에는 heading과 Header만 보이고 수정 card는 표시되지 않는다.

## 2. Header

- 데스크톱 높이 78px, 모바일 70px
- 어두운 반투명 배경, blur와 하단 border
- 좌측 원형 뒤로가기, 중앙 로고, 우측 프로필
- 뒤로가기는 현재 상세, 로고는 목록으로 이동

## 3. Heading과 card

- 본문 최대 너비 760px, 중앙 정렬
- `Edit Post`는 민트색 13px 굵은 글자
- `게시글 수정`은 34px, 모바일 28px
- 설명은 흐린 보조색 15px
- card는 어두운 반투명 배경, 14px radius, blur와 shadow
- 데스크톱 padding 34px 38px 38px, 모바일 28px 20px 30px

## 4. 입력과 helper

- label은 시각적으로 표시되는 15px 굵은 글자
- input 높이 50px, textarea 최소 높이 260px
- focus 시 민트 border와 focus ring
- helper는 최소 18px 높이, 붉은색 12px
- 제목 placeholder: `제목을 입력해주세요. (최대 26글자)`
- 내용 placeholder: `내용을 입력해주세요.`

## 5. 이미지

- 기존·새 이미지 모두 정사각형 `object-fit: cover`
- 데스크톱 3열, 640px 이하 2열 grid
- 새 이미지 file input은 민트 점선 border와 전용 선택 버튼 스타일
- 미선택 문구: `새 이미지를 선택하지 않으면 기존 이미지가 유지됩니다.`
- 선택 문구: `{파일명들} 선택됨 - 기존 이미지를 교체합니다.`

## 6. 수정 버튼

- card 하단 중앙에 배치
- 너비 360px, 최대 100%, 높이 48px
- 민트 gradient와 어두운 글자
- 비활성 상태는 opacity 0.55와 금지 cursor
- 요청 중에도 문구는 `수정하기`로 유지

## 7. 반응형

640px 이하에서 Header control, main padding, card padding과 제목이 축소되고 이미지 grid가 2열이 된다.
