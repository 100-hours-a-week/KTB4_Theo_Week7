# 게시글 작성 텍스트 UI 기준선

캡처 없이 Vanilla와 React 화면을 비교하기 위한 기준이다.

## 1. 전체 배치

```text
┌──────────────────────────────────────────────────────────────┐
│ [←]                    </> 코드 한입             [프로필]   │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ 게시글 작성                                                  │
│                                                              │
│ 오늘의 개발 한입을 적어주세요____________________________    │
│ [제목 helper 영역]                                           │
│                                                              │
│ 배운 점, 막혔던 부분, 해결한 과정을 자유롭게 남겨보세요.     │
│                                                              │
│ [내용 helper 영역]                                           │
│                                                              │
│ ┌ 이미지 ────────────────────────────────────────────────┐   │
│ │ [파일 선택]                                            │   │
│ │ 선택된 이미지가 없습니다.                              │   │
│ │ [미리보기 grid]                                        │   │
│ └────────────────────────────────────────────────────────┘   │
│                                                              │
├──────────────────── 화면 하단 고정 ──────────────────────────┤
│                                                [완료]        │
└──────────────────────────────────────────────────────────────┘
```

## 2. Header 기준

- 높이: 데스크톱 78px, 760px 이하 70px
- 반투명 어두운 배경, blur와 하단 border
- 뒤로가기: 좌측 원형 42px, 모바일 36px
- 로고: 중앙 `</>` 민트색, `코드 한입` 흰색
- 프로필: 우측 원형 42px, 모바일 36px
- 프로필 메뉴는 공통 Header의 어두운 드롭다운 스타일

## 3. Editor 기준

- 전체 배경은 로그인 배경 이미지 위에 어두운 gradient를 적용한다.
- 본문 최대 너비는 1080px이고 중앙 정렬한다.
- 페이지 제목은 24px, 굵기 800이다.
- 제목 input은 투명 배경과 두꺼운 하단 border를 사용한다.
- 제목 글자 크기는 viewport에 따라 약 34~56px이며 focus 시 border가 민트색이다.
- 내용 textarea는 데스크톱 최소 높이 430px, 23px 글자와 넓은 줄 간격을 사용한다.
- helper는 최소 20px 높이를 유지하고 붉은색 13px 굵은 글자로 표시한다.

## 4. 이미지 영역 기준

- 최대 너비 720px의 어두운 반투명 card다.
- file input은 민트색 점선 border와 전용 파일 선택 버튼 스타일을 가진다.
- 선택 파일명은 흐린 보조색이며 긴 이름은 줄바꿈할 수 있다.
- 미리보기는 정사각형, `object-fit: cover`다.
- 데스크톱은 4열, 760px 이하는 2열 grid다.

## 5. 완료 action bar

- viewport 하단에 고정되며 전체 너비를 사용한다.
- 버튼은 우측에 배치되고 데스크톱 최소 164×52px다.
- 활성 버튼은 민트색 gradient, 어두운 글자와 그림자를 가진다.
- 비활성 버튼은 opacity가 낮고 금지 cursor이며 그림자가 없다.

## 6. 상태별 텍스트

| 상태 | 표시 텍스트 |
|---|---|
| 초기 이미지 | `선택된 이미지가 없습니다.` |
| 제목 빈 값 | `* 제목을 입력해주세요.` |
| 제목 길이 초과 | `* 제목은 최대 26자까지 작성 가능합니다.` |
| 내용 빈 값 | `* 내용을 입력해주세요.` |
| submit 기본 문구 | `완료` |

요청 중 버튼 문구는 변경하지 않고 `완료`를 유지한다.

## 7. 반응형 기준

### 760px 이하

- main 좌우 padding 18px
- 제목 input 32px, 내용 textarea 19px·최소 높이 360px
- 이미지 미리보기 2열
- action bar 높이 78px, 완료 버튼 최소 126×48px

### 520px 이하

- 중앙 로고 글자 20px, `</>` 24px
