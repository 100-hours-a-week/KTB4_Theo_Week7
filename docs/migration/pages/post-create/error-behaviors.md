# 게시글 작성 오류 상태별 UI 동작

## 1. 클라이언트 검증

| 조건 | API 요청 | UI 동작 |
|---|---|---|
| 제목이 비어 있거나 공백뿐이다 | 전송 안 함 | 제목 helper에 `* 제목을 입력해주세요.` |
| 제목이 trim 후 26자를 초과한다 | 전송 안 함 | 제목 helper에 `* 제목은 최대 26자까지 작성 가능합니다.` |
| 내용이 비어 있거나 공백뿐이다 | 전송 안 함 | 내용 helper에 `* 내용을 입력해주세요.` |
| 하나 이상의 필드가 유효하지 않다 | 전송 안 함 | 완료 버튼 비활성화 |
| 작성 요청 중 다시 제출한다 | 추가 전송 안 함 | 완료 버튼 비활성화 상태 유지 |

HTML `maxlength=26` 때문에 일반 입력으로 26자를 초과하기 어렵지만, 검증 함수와 서버 오류 처리는 별도로 유지한다.

## 2. POST `/posts` 오류

오류를 처리하기 전에 제목·내용의 기존 서버 helper를 모두 지우고 아래에서 일치하는 항목 하나를 표시한다.

| Status | message | UI 동작 | 입력 상태 | 이동 |
|---:|---|---|---|---|
| 400 | `blank_title` | 제목 helper에 `* 제목을 입력해주세요.` | 유지 | 없음 |
| 400 | `blank_content` | 내용 helper에 `* 내용을 입력해주세요.` | 유지 | 없음 |
| 400 | `invalid_post_title` | 제목 helper에 `* 제목은 최대 26자까지 작성 가능합니다.` | 유지 | 없음 |
| 401 | `unauthorized_request` | `로그인이 필요합니다.` alert | 페이지 이탈 | `/login` |
| 429 | `post_create_rate_limit_exceeded` | `게시글은 1분에 최대 3개까지 작성할 수 있습니다.` alert | 유지 | 없음 |
| 500 | `internal_server_error` | `서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.` alert | 유지 | 없음 |
| 기타 status/message | 기타 | `게시글 작성 중 오류가 발생했습니다. 다시 시도해주세요.` alert | 유지 | 없음 |
| 네트워크·응답 파싱 오류 | 없음 | 일반 작성 오류 alert | 유지 | 없음 |

이동하지 않는 오류에서는 요청 상태를 해제한 뒤 제목·내용 유효성에 따라 완료 버튼을 다시 활성화한다.

## 3. 인증 초기화와 Header 오류

작성 Route는 `ProtectedRoute`의 기존 정책을 그대로 따른다.

- Access Token이 없으면 Refresh Token 쿠키로 재발급을 시도한다.
- 재발급 또는 현재 사용자 조회에 실패해 인증되지 않으면 `/login`으로 이동한다.
- 로그아웃 실패 시 공통 Header의 기존 오류 alert를 표시하고 현재 화면을 유지한다.

## 4. 이미지 관련 현재 동작

- 파일 선택 취소: 파일명 문구와 미리보기를 초기화하며 오류 helper는 없다.
- 미리보기 로드 실패: 별도 fallback이나 오류 문구 없이 깨진 이미지 상태가 될 수 있다.
- 파일 형식·크기·개수 초과: 프론트엔드 전용 오류 처리가 없다.
- Object URL 생성 실패: Vanilla에는 별도 처리 로직이 없다.

React 1차 구현에서 새로운 제한을 임의로 추가하지 않는다. 오류 fallback을 개선한다면 구현 변경 기록과 체크리스트 기대 결과를 함께 수정한다.
