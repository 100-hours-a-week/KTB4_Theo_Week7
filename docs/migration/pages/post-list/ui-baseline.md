# 게시글 목록 텍스트 UI 기준선

이 문서는 캡처 대신 게시글 목록의 구조, 문구, 카드 상태와 반응형 배치를 기록한다.

## 1. UI-POST-LIST-01 기본 구조

```text
┌──────────────────────────────────────────────────────────┐
│                 </> 코드 한입             (프로필 이미지)│
├──────────────────────────────────────────────────────────┤
│                                                          │
│ 전체 피드  // 개발자들의 지식과 경험을 한입에  [게시글 작성]│
│                                                          │
│ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐             │
│ │게시글 1│ │게시글 2│ │게시글 3│ │게시글 4│             │
│ └────────┘ └────────┘ └────────┘ └────────┘             │
│                                                          │
│                    (무한 스크롤 sentinel)                 │
└──────────────────────────────────────────────────────────┘
```

- 어두운 배경 이미지와 반투명 Header를 사용한다.
- Header 높이는 기본 78px이다.
- 콘텐츠 최대 너비는 1,480px이며 기본 화면에서 4열 카드다.

## 2. UI-POST-LIST-02 게시글 카드

```text
┌──────────────────────────────────┐
│ 게시글 제목               [수정됨]│
│                                  │
│ ♥ 12  댓글 3  조회수 100  2026.07.19│
├──────────────────────────────────┤
│ (작성자 이미지) 작성자 닉네임     │
└──────────────────────────────────┘
```

- 카드 최소 높이: 216px
- 제목: 최대 26자 데이터, 화면에서는 최대 두 줄
- `수정됨`: `edited`일 때만 표시
- 좋아요, 댓글, 조회수, 날짜는 카드 본문 하단에 배치
- 작성자 영역은 구분선 아래에 표시
- hover 시 테두리와 그림자가 강조되고 2px 위로 이동

## 3. UI-POST-LIST-03 수치 표시

```text
999   → 999
1000  → 1k
1999  → 1k
10000 → 10k
null 또는 숫자 변환 실패 → 0
```

## 4. UI-POST-LIST-04 작성자 예외

```text
프로필 이미지 없음 → 기본 프로필 이미지
프로필 이미지 로드 실패 → 기본 프로필 이미지
authorDeleted=true → 닉네임 "알 수 없음"
```

프로필 이미지는 장식 목적이므로 카드 내부 이미지의 alt는 빈 문자열이다.

## 5. UI-POST-LIST-05 숨김 게시글

```text
blinded=false → 카드 표시
blinded=true  → 카드 전체 미표시
```

`숨김 처리된 게시글입니다.`라는 제목 변환 함수가 존재하지만 현재 렌더링 단계에서 숨김 게시글을 먼저 제외하므로 목록에는 문구가 나타나지 않는다.

## 6. UI-POST-LIST-06 프로필 메뉴 닫힘

```text
(프로필 이미지 버튼)
메뉴: hidden
aria-expanded="false"
```

## 7. UI-POST-LIST-07 프로필 메뉴 열림

```text
┌────────────────┐
│ 회원정보 수정  │
│ 비밀번호 수정  │
│ 로그아웃       │
└────────────────┘
aria-expanded="true"
```

외부 클릭 또는 Escape로 닫히며 Escape일 때 버튼으로 포커스가 돌아간다.

## 8. UI-POST-LIST-08 로딩·빈 목록·오류

```text
최초 로딩: 별도 spinner·문구 없음, 카드 영역이 비어 있음
추가 로딩: 기존 카드 유지, 별도 spinner·문구 없음
빈 목록: 별도 안내 없음
오류: 브라우저 alert, 카드 영역 자체의 오류 UI 없음
마지막 페이지: 별도 종료 문구 없음
```

## 9. 반응형 기준

| viewport | 카드 열 | 주요 변화 |
|---:|---:|---|
| 1240px 초과 | 4열 | 기본 배치 |
| 1240px 이하 | 3열 | 카드 열 감소 |
| 760px 이하 | 2열 | Header 70px, 인사말 세로 배치, 작성 버튼 왼쪽 배치 |
| 520px 이하 | 1열 | 프로필 버튼 36px, 작은 로고, 카드 정보 줄바꿈 |

## 10. React 비교 판정 기준

- 같은 데이터에서 카드 수와 순서가 일치한다.
- 제목, 배지, 수치, 날짜, 작성자 fallback이 일치한다.
- 숨김 게시글이 표시되지 않는다.
- 추가 페이지가 기존 목록 뒤에 이어진다.
- 동일 breakpoint에서 열 수와 Header·작성 버튼 배치가 일치한다.
- 프로필 메뉴의 시각 상태와 키보드 동작이 일치한다.
- 로딩·빈 목록·오류 UI를 임의로 추가했다면 설계 변경으로 기록한다.
