# 회원가입 오류 상태별 화면 동작

## 1. 클라이언트 검증 오류

| 대상 | 조건 | 표시 내용 |
|---|---|---|
| 프로필 이미지 | 선택 파일 없음 | `* 프로필 사진을 추가해주세요.` |
| 이메일 | 빈 값 | `* 이메일을 입력해주세요.` |
| 이메일 | 형식 오류 | `* 올바른 이메일 주소 형식을 입력해주세요. (예: example@example.com)` |
| 비밀번호 | 빈 값 | `* 비밀번호를 입력해주세요.` |
| 비밀번호 | 형식 오류 | 8~20자 및 대·소문자, 숫자, 특수문자 규칙 안내 |
| 비밀번호 확인 | 빈 값 | `* 비밀번호를 한번 더 입력해주세요.` |
| 비밀번호 확인 | 비밀번호와 불일치 | `* 비밀번호가 다릅니다.` |
| 닉네임 | 빈 값 | `* 닉네임을 입력해주세요.` |
| 닉네임 | 내부 공백 포함 | `* 띄어쓰기를 없애주세요.` |
| 닉네임 | 10자 초과 | `* 닉네임은 최대 10자까지 작성 가능합니다.` |

클라이언트 검증에 실패하면 API 요청을 보내지 않고 버튼을 비활성화한다.

## 2. API 오류 공통 처리

API 오류를 처리하기 직전에 이메일, 비밀번호, 비밀번호 확인, 닉네임 helper를 모두 비운다. 프로필 이미지 helper는 이 초기화 대상에 포함되지 않는다.

## 3. 상태와 message별 동작

| HTTP 상태 | `message` | UI 동작 | 이동 |
|---:|---|---|---|
| 400 | `invalid_request` | `입력값을 다시 확인해주세요.` alert | 없음 |
| 400 | `invalid_email_format` | 이메일 helper에 `* 올바른 이메일 주소 형식을 입력해주세요.` | 없음 |
| 400 | `invalid_password_format` | 비밀번호 helper에 형식 안내 | 없음 |
| 400 | `invalid_nickname_format` | 닉네임 helper에 띄어쓰기 없이 10자 이내 안내 | 없음 |
| 400 | 그 외 | `입력값을 다시 확인해주세요.` alert | 없음 |
| 409 | `email_already_exist` | 이메일 helper에 `* 중복된 이메일입니다.` | 없음 |
| 409 | `nickname_already_exist` | 닉네임 helper에 `* 중복된 닉네임입니다.` | 없음 |
| 409 | 그 외 | `이미 사용 중인 정보가 있습니다.` alert | 없음 |
| 500 | 구분하지 않음 | `서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.` alert | 없음 |
| 그 외 | 구분하지 않음 | `회원가입 중 오류가 발생했습니다. 다시 시도해주세요.` alert | 없음 |

## 4. 네트워크 및 응답 처리 오류

다음 오류는 status가 없는 일반 오류로 처리한다.

- 네트워크 연결 실패
- CORS 차단
- JSON으로 파싱할 수 없는 실패 응답

표시 문구:

```text
회원가입 중 오류가 발생했습니다. 다시 시도해주세요.
```

## 5. 오류 후 상태

- 사용자가 입력한 값은 유지한다.
- 선택한 프로필 이미지와 미리보기는 유지한다.
- 요청이 끝나면 `isSignupSubmitting`을 false로 복구한다.
- 폼이 계속 유효하면 버튼을 다시 활성화한다.
- 서버 오류 처리 전에 기존 필드 helper를 초기화하므로 현재 서버 오류와 관계없는 이전 helper는 제거된다.

## 6. React 이전 주의사항

- 409 중복 오류를 전역 오류 화면으로 보내지 않고 해당 필드 helper로 표시한다.
- 로그인과 회원가입의 이메일 형식 helper 예시 문구가 서로 다르므로 동일 상수로 무조건 합치지 않는다.
- API 오류 시 프로필 이미지 helper를 임의로 지우지 않는다.
- 성공·실패 후 폼 값을 자동 초기화하지 않는다.
- alert를 Toast로 변경하는 UX 개선은 동등성 검증 이후 별도 작업으로 처리한다.
