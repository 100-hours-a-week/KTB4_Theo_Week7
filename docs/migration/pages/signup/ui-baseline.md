# 회원가입 텍스트 UI 기준선

이 문서는 캡처 대신 회원가입 화면의 구조·문구·상태를 텍스트로 기록한다.

## 1. 공통 화면 구조

```text
┌─────────────────────────────────────────────┐
│ </> 코드 한입                               │
│ 개발자의 성장 한입, 지식의 공유 한입       │
│                                             │
│ [뒤로가기]                                  │
│ 회원가입                                    │
│ 코드 한입에서 지식을 나누고 성장 기록을    │
│ 쌓아보세요.                                 │
│                                             │
│ 프로필 사진*                                │
│ helper                                      │
│            (기본 프로필 아이콘)             │
│                                             │
│ 이메일*                                     │
│ [이메일을 입력하세요]                       │
│                                             │
│ 비밀번호*                                   │
│ 8~20자, 대·소문자·숫자·특수문자 안내       │
│ [비밀번호를 입력하세요]                     │
│                                             │
│ 비밀번호 확인*                              │
│ [비밀번호를 한번 더 입력하세요]             │
│                                             │
│ 닉네임*                                     │
│ 띄어쓰기 없이 10자 이내 안내                │
│ [닉네임을 입력하세요]                       │
│                                             │
│ [비활성: 회원가입]                          │
│ 이미 계정이 있으신가요? 로그인              │
└─────────────────────────────────────────────┘
```

## 2. UI-SIGNUP-01 최초 상태

```text
프로필: 기본 아이콘, 미리보기 숨김, helper 없음
이메일: 빈 값, helper 없음
비밀번호: 빈 값, helper 없음
비밀번호 확인: 빈 값, helper 없음
닉네임: 빈 값, helper 없음
[비활성: 회원가입]
```

## 3. UI-SIGNUP-02 프로필 이미지 미선택

```text
프로필 사진*
helper: * 프로필 사진을 추가해주세요.
(기본 프로필 아이콘 표시)
[비활성: 회원가입]
```

## 4. UI-SIGNUP-03 프로필 이미지 선택

```text
프로필 사진*
helper: 표시 안 함
(선택 이미지가 원형 영역 전체에 미리보기로 표시)
(기본 프로필 아이콘 숨김)
```

선택 이미지 영역을 다시 선택하면 이미지가 제거되고 `UI-SIGNUP-02`로 돌아간다.

## 5. UI-SIGNUP-04 이메일 오류

```text
빈 값:
helper: * 이메일을 입력해주세요.

형식 오류:
helper: * 올바른 이메일 주소 형식을 입력해주세요. (예: example@example.com)
```

## 6. UI-SIGNUP-05 비밀번호 오류

```text
빈 값:
helper: * 비밀번호를 입력해주세요.

형식 오류:
helper: * 비밀번호는 8자 이상, 20자 이하이며, 대문자, 소문자, 숫자,
        특수문자를 각각 최소 1개 포함해야 합니다.
```

## 7. UI-SIGNUP-06 비밀번호 확인 오류

```text
빈 값:
helper: * 비밀번호를 한번 더 입력해주세요.

불일치:
helper: * 비밀번호가 다릅니다.
```

비밀번호 필드가 blur되어도 비밀번호 확인 helper를 다시 계산한다.

## 8. UI-SIGNUP-07 닉네임 오류

```text
빈 값:
helper: * 닉네임을 입력해주세요.

내부 공백:
helper: * 띄어쓰기를 없애주세요.

10자 초과:
helper: * 닉네임은 최대 10자까지 작성 가능합니다.
```

## 9. UI-SIGNUP-08 유효한 폼

```text
프로필: 이미지 선택됨
이메일: 유효
비밀번호: 유효
비밀번호 확인: 비밀번호와 일치
닉네임: 유효
모든 helper: 표시 안 함
[활성: 회원가입]
```

## 10. UI-SIGNUP-09 요청 중

```text
모든 입력값과 이미지 미리보기 유지
[비활성: 회원가입]
별도의 spinner 또는 로딩 문구 없음
추가 제출 무시
```

## 11. UI-SIGNUP-10 이메일 중복

```text
이메일 입력값 유지
email helper: * 중복된 이메일입니다.
다른 필드 입력값과 이미지 유지
```

## 12. UI-SIGNUP-11 닉네임 중복

```text
닉네임 입력값 유지
nickname helper: * 중복된 닉네임입니다.
다른 필드 입력값과 이미지 유지
```

## 13. UI-SIGNUP-12 성공

```text
alert: 회원가입이 완료되었습니다.
사용자가 alert를 닫음
→ 기존: pages/login.html
→ React: /login
```

## 14. React 비교 판정 기준

- 동일 조건에서 같은 필드 아래에 같은 helper가 표시된다.
- 프로필 이미지 선택·미리보기·제거 동작이 일치한다.
- 유효한 폼과 요청 중 버튼 상태가 일치한다.
- 실패 후 입력값과 선택 이미지가 유지된다.
- 400·409·500 및 기타 오류 피드백이 기존과 일치한다.
- 성공 alert 후 `/login`으로 이동한다.
- 기존 CSS 클래스와 자산을 사용해 레이아웃을 보존한다.
